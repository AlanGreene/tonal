Work with scales.

A tonal tutorial

Creating scales

The scale function from music-scale package is the primary function to create scales. This function is available in tonal as tonal.scale

Create scale using type and tonic

This is the basic usage. If tonic is a pitch class, the scale notes will be pitch classes:


          var scale = require('music-scale')
          scale('major', 'A')
          // => ['A', 'B', 'C#', 'D', 'E', 'F#', 'G#']

          // or using tonal
          tonal.scale('major', 'A')
          // => ['A', 'B', 'C#', 'D', 'E', 'F#', 'G#']

If you use a note as a tonic, properly octaved notes are returned:


          tonal.scale('major', 'A4')
          // => ['A4', 'B4', 'C#5', 'D5', 'E5', 'F#5', 'G#5']

Create scale using intervals and tonic

The scale function accepts a list of intervals as first parameter:


          tonal.scale('1P 2M 3M 4P 5P 6M 7m 8P', 'A4')
          // => ['A4', 'B4', 'C#5', 'D5', 'E5', 'F#5', 'G5', 'A5']

Again, you can use pitch classes:


          tonal.scale('1P 2M 3M 4P 5P 6M 7m 8P', 'A')
          // => ['A', 'B', 'C#', 'D', 'E', 'F#', 'G', 'A']

Get scale intervals

Use false as tonic to get scale intervals:


          tonal.scale('major', false)
          // => ['P1', 'M2', 'M3', 'P4', 'P5', 'M6', 'M7']

Get scale from scale name

scale.get is a convenient function to create scales using the complete scale name (instead to type and tonic):


          tonal.scale.get('A major')
          // => ['A', 'B', 'C#', 'D', 'E', 'F#', 'G#']

Get available scale names

The scale.names function returns all the scale names:


          tonal.scale.names()
          // => ['major', 'minor', ...]
          tonal.scale.names(true)
          // => ['major', 'ionian', 'minor', 'aeolian', ...]
